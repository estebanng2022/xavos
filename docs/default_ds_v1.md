# Xavos — Design System por defecto (v1)

## Objetivo
Definir un Design System base que Xavos usará cuando el usuario no tenga uno propio.

## Alcance del DS por defecto
- Colores base (light/dark) — pendientes de definir.
- Tipografía base — pendiente de definir.
- Componentes mínimos:
  - Button
  - TextField
  - AppBar
  - Card
  - List item

## Reglas
- El DS por defecto debe ser:
  - simple
  - limpio
  - fácil de extender
- Nunca se mezcla con DS personalizados del cliente.
- Si el proyecto define su propio DS, Xavos usa el del proyecto y no el DS por defecto.

## Estado actual
- DS por defecto: solo definido conceptualmente.
- Colores, tipografía y tokens se definirán en una fase posterior.
